# よみもち 開発ガイド

子ども向け読書記録アプリ。バニラ HTML/CSS/JS の1ページ構成・ビルドなし・サーバーなし。
このファイルは「どのモデル・どの開発者が触っても壊さない」ための技術ガイド。

## ファイル構成（この4つがすべて）

| ファイル | 役割 |
|---|---|
| index.html | 全ページのDOM（`.page` を nav() で出し分け）とモーダル |
| app.js | 全ロジック。冒頭にセクション目次あり。まずそれを読む |
| styles.css | 見た目。CSS変数でテーマ色を定義 |
| config.js | デプロイ先URL・Amazonアソシエイト設定。**ロジックを書かない** |

## アーキテクチャ（3行）

1. 状態はグローバル `db`（profiles配列＋currentId）ただ1つ。`P()` が「いま選択中の子」を返し、全機能がそれ経由
2. 変更の作法: `db` をいじる → `save()` → 関係する `render○○()` を呼ぶ。これ以外の状態管理はない
3. データは `localStorage` のみ。サーバー通信は書籍検索（openBD/Google Books）だけ

## 壊してはいけない不変条件（変更前に必ず確認）

1. **`STORE = 'yomiyomi.v1'` を変えない**。旧名時代からのキーで、変えると既存ユーザーの記録が全部消える
2. **データ構造を変えるときは `load()` に移行処理を足す**。過去形式のデータを読んだら新形式に変換して引き継ぐ（`load()` 内に単一ユーザー→複数プロフィールの移行例あり）
3. **ユーザー入力をHTMLに埋めるときは必ず `esc()` を通す**（XSS防止。本のタイトル・子の名前・感想すべて）
4. **子ども画面に広告・課金・外部リンクを置かない**。お金にまつわる導線は「おうちの人へ」ページ（`page-parent`）だけ
5. **子ども向けUI原則**: ひらがな中心・大きなボタン・漢字を使わない（保護者ページのみ漢字可）
6. **バックアップの後方互換**: `importData()` は旧形式JSONも受け付ける。取り込み条件を厳しくしない

## 環境の落とし穴（ハマりどころ）

- **iOS/Safari系は `BarcodeDetector` 非対応**（全iOSブラウザ共通）。スキャナーは対応環境でのみ動く設計。iOSアプリ化ではネイティブ差し替えが必要
- **音声入力（Web Speech）はWKWebView/PWA内で動かない**。Safari本体では動く
- マスコットは `mochiSVG(mood,size)` がインラインSVGを生成（クレヨン風はfeTurbulenceフィルタ2段）。**フィルタidは連番生成**にしてある——固定idにするとページ内で衝突して描画が乱れる

## 動作確認のしかた（このプロジェクトの流儀）

1. 構文・結線の確認: `python3 -m http.server 8000 --bind 127.0.0.1` で配信し、curl＋grepで要素ID・関数定義の存在を確認
2. 見た目の確認: ヘッドレスChromeでスクリーンショット
   `"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" --headless --screenshot=out.png --window-size=390,844 file://$PWD/index.html`
3. SVG単体の確認: `qlmanage -t -s 1280 -o <出力先> file.svg` でPNG化して目視
4. OGP画像の再生成（ogp.svg を変えたら必ず）: 手順2と同じChromeで `--window-size=1200,630` で ogp.svg を撮影し ogp.png を上書き

## 変更後のチェックリスト

- [ ] 既存の localStorage データを消す変更をしていない（不変条件1・2）
- [ ] 新しいユーザー入力表示に `esc()` を通した
- [ ] 子ども画面に外部リンク・お金の導線を増やしていない
- [ ] ogp.svg を変えたら ogp.png も再生成した
- [ ] ヘッドレスChromeでスクリーンショットを撮り、表示が崩れていないことを確認した
